# Task
请帮我完成 VPS 上的代理搭建，通过指定的住宅 IP 出口，并在 VPS 上搭建代理订阅服务。最后通过终端命令自动唤起我本地的客户端（Clash 或 Shadowrocket）注入订阅，并通过环境变量验证代理出局 IP。整个过程由你通过本地终端和 SSH 直接执行，我只负责提供信息和审核授权。

# Anti-Hallucination Rules (【最高优先级】防幻觉与真实执行强制规范)

2. **强制前置检查**：在修改任何文件前，必须先真实执行 `ls`, `find` 或 `cat` 命令确认文件真实存在及当前内容，**绝不能靠猜测路径**。
3. **强制后置验证**：每执行完一个关键命令，必须紧接着执行验证命令（如 `systemctl status`, `curl` 测试），只有读取到真实的成功日志后，才能向我汇报“已完成”。
4. **提供真实证据**：每次向我简报进度时，必须附带你刚刚真实获取的终端输出片段或日志作为证据。

# Architecture
用户设备 (本地客户端拉取订阅) -> VPS (轻量 HTTP 订阅服务 + Gost SS 服务端转发) -> 住宅 IP (Socks5) -> 目标网站

# Execution Steps

## 第一阶段：信息收集
请先向我索要以下信息，待我提供完毕后再进入第二阶段：
- VPS IP 及 SSH 端口
- VPS root 密码（或当前可用的 SSH 登录方式）
- 新用户名（默认 proxyuser）
- 住宅 IP 及端口
- 住宅 IP 认证信息（用户名/密码，如有）

## 第二阶段：VPS 基础安全与环境配置（通过 SSH 真实执行）
1. **创建新用户**：使用 root 权限创建新用户并加入 sudo 组。执行后用 `id 新用户名` 验证。
   - **【权限切换警告】**：从这一步之后，VPS 上的所有搭建和配置工作，**必须优先使用这个新用户（配合 `sudo` 命令）来执行**。只有在实在没有权限且 `sudo` 也无法解决的情况下，才允许回退使用 root 账号。
2. **SSH 安全配置（防锁死警告）**：
   - 检查我本地是否有 SSH 密钥，如果没有则真实调用 `ssh-keygen` 生成。
   - 将公钥配置到 VPS 的 root 和新用户下。
   - **【必须暂停】**：提示我新开一个终端手动测试密钥登录是否成功。
   - **【彻底禁用密码登录】**：只有我回复“测试成功”后，你才能真实去修改 `/etc/ssh/sshd_config` 文件，**明确禁用 root 和新用户的密码登录（确保设置 `PasswordAuthentication no` 以及 `PermitRootLogin prohibit-password` 或 `no`）**，然后重启 SSH 服务。
3. **防火墙配置**：使用新用户自动检测 VPS 的防火墙，放行 SSH 端口及后续生成的代理端口和订阅服务端口。执行后用 `sudo ufw status` 或 `sudo iptables -L` 验证。
4. **网络优化**：使用新用户自动修改 sysctl 开启 BBR。执行后用 `lsmod | grep bbr` 验证。

## 第三阶段：核心代理与订阅服务搭建（通过 SSH 使用新用户真实执行）
1. **安装与配置 Gost**：
   - 下载最新版 Gost 二进制文件并放置到系统 PATH 中。
   - 随机生成高强度加密方式（如 aes-256-gcm）和随机端口。
   - 写入 Systemd `.service` 文件，配置 Gost 监听 SS 协议并转发至住宅 IP，设置开机自启并启动。验证状态为 active。
2. **生成订阅配置文件**：
   - 在 VPS 上创建一个专用目录（如 `/opt/proxy-sub/`）。
   - 根据前面生成的 SS 节点信息，结合 `https://github.com/Loyalsoldier/clash-rules` 规则，在 VPS 上生成一份完整的 YAML 配置文件（兼容 Clash 和 Shadowrocket）。
3. **搭建轻量订阅服务**：
   - 使用轻量级方案（如 Python3 的 `http.server` 或 Nginx）在 VPS 上随机分配一个端口（如 8080 或其他），将上一步生成的 YAML 文件作为静态文件暴露出去。
   - 为该 HTTP 服务编写 Systemd 守护进程，确保开机自启。
   - 确保防火墙已放行该订阅端口。
   - 最终生成一个订阅链接（如 `http://VPS_IP:端口/config.yaml`）。

## 第四阶段：本地端测试与交付（在本地终端真实执行）
1. **自动唤起客户端并注入订阅（【必须暂停确认】）**：
   - 询问我本地使用的是哪款代理客户端（如 Clash 还是 Shadowrocket）。
   - 根据我的回答，使用你刚刚搭建好的订阅链接拼接一键导入协议：
     - **如果是 Clash**：将订阅链接进行 URL Encode 编码，拼接为 `clash://install-config?url=编码后的订阅链接&name=VPS_Proxy`。
     - **如果是 Shadowrocket**：将订阅链接进行 Base64 编码，拼接为 `shadowrocket://add/sub://Base64编码后的订阅链接?title=VPS_Proxy`。
   - 自动检测我本地的操作系统，并真实执行唤起命令：
     - macOS: `open "协议链接"`
     - Windows: `start "协议链接"`
     - Linux: `xdg-open "协议链接"`
   - **【必须暂停】**：向我确认本地客户端是否已成功弹出并导入了配置。同时询问我本地代理软件的混合代理端口（Mixed Port，Clash 通常是 7890，Shadowrocket 通常是 1082 等）。等待我回复确认后，再进行下一步。
2. **本地终端环境变量测试**：
   - 自动检测我本地终端的操作系统和 Shell 类型（Windows CMD / Windows PowerShell / macOS zsh / Linux bash 等）。
   - 根据不同的 Shell 类型，**真实执行**临时增加环境变量的命令，让当前终端走代理。
     - *示例 (Unix/macOS)*: `export http_proxy=http://127.0.0.1:端口; export https_proxy=http://127.0.0.1:端口`
     - *示例 (PowerShell)*: `$env:http_proxy="http://127.0.0.1:端口"; $env:https_proxy="http://127.0.0.1:端口"`
   - 紧接着在同一个终端会话中真实执行 `curl -s https://ipinfo.io`。
   - 读取真实返回的 IP 地址，并与第一阶段我提供的住宅 IP 进行比对，验证出局 IP 是否成功。
